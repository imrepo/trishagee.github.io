New Gradle Project

File -> New Project Gradle. Tick create content roots

Gradle will build (bottom status bar) When this is done, you’ll get your source/main/java / test

If you open build.gradle, you might get the suggestion to add the sources.
Apply this suggestion, it changes gradle-wrapper.properties to get gradle to download sources as well as jar files. Super useful for debugging (althugh without sources intellij’s decompiler will give you an approsimation)

Open the gradle window, and you can see the tasks you can run. By default you’ll get a gradle file with the java plugin, and this will give you the tasks you expect for a java project - clean, build, test, etc.

Open “verification” and run check - this would compile any classes and run any tests. THere’s nothing to do for this project as it’s new, but you should get a Build Successful in the Run window, showing that gradle is configured correctly.

Gradle painlessly handles your project dependencies. By default, IntelliJ has included JUnit. You can see IntelliJ puts the downloaded dependencies under external depdencies in the Project window.

You can change your gradle settings by pressing the settings button in the gradle window. For example, you might want to enable offline mode, if you do work in places without internet connection (planes and trains for example).

Adding more dependencies

You might prefer the shorter notation for dependencies, so I’ve changed the default JUnit dep to use this notation. Add new dependencies into the dependencies section of the build file:

(e.g. guava)

When you’ve done this, you need to get IntelliJ to fetch the new dependency. Do this by clicking “refresh all gradle projects”

You should see the new depdency in the external libraries.

Note - gradle dependencies will appear in the project dependencies, but if you add a dependency in here it will not be added to the gradle build. Any dependencies here will be lost on sync?
