Java 8

I’ve been giving a presentation showing “Java 8 in Anger” - no, it’s not that the most recent release of Java has me particularly enraged, “in anger” is (apparently) a British phrase meaning “in practice” or “in the real world”.

The aim of this talk is to demo, with live code, how to use some of the Java 8 features like Lambda Expressions and the Streams API to solve some of the coding problems you might come across in your day job as a developer.

I’m not going to transcribe the whole talk for this article, it probably works best (if you’re interested) for you to watch the video instead. What I do want to do is to take the time to explore some specific sections in a bit more detail. If you’re interested in the wider context, I’ve got links to the videos and source code on my blog.

The first section where we start using some of the new methods available in Java 8 is when we’re building up a leaderboard of top tweeters. We store every Twitter handle we see in a map, and for the value associated with that key, we have a TwitterUser (an object to represent this particular user). Now, when we see a new Twitter handle, we want to create a new TwitterUser for this person and store it in the map, keyed against this Twitter handle.

Pre-Java-8, we’d write something like this to do this:

private final Map<String, TwitterUser> allTwitterUsers = new HashMap<>();

@Override
public void onMessage(String twitterHandle) {
    TwitterUser twitterUser = allTwitterUsers.get(twitterHandle);
    if (twitterUser == null) {
        twitterUser = new TwitterUser(twitterHandle);
        allTwitterUsers.put(twitterHandle, twitterUser);
    }
    // then do stuff...
}

But Java 8 has not only given us lambdas and streams, it’s added new methods to some of our favourite classes. Map now has a computeIfAbsent method which allows you to define what to do if an item doesn’t exist in the Map:

private final Map<String, TwitterUser> allTwitterUsers = new HashMap<>();

@Override
public void onMessage(String twitterHandle) {
    TwitterUser twitterUser = allTwitterUsers.computeIfAbsent(twitterHandle, TwitterUser::new);
    // then do stuff...
}

This is clearly far fewer lines of code, especially as it also makes use of Method References, which usually work out to be even shorter than lambda expressions. Now, I don’t know about you, but as a long-term Java programmer with limited exposure to other languages, while this may be shorter, learning to write code this way is going to take a bit of getting used to.

Fortunately for me and my old-school Java approach, IntelliJ IDEA can help me get to grips with the new syntax. When I first came to Java 8, I found it more understandable (although much more code) to call these sorts of methods using an anonymous inner class rather than a lambda expression or a method reference.
